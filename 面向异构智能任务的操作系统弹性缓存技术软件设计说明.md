## 1.2 系统概述

### 1.2.1 需求分析

随着智能服务从单一模型推理逐步发展为多模型、多平台、多负载条件下的异构智能任务执行，操作系统需要在有限算力、有限内存、有限网络带宽以及动态任务到达的约束下，为智能任务提供稳定、高效、可调度的运行支撑。异构智能任务通常具有模型规模差异大、启动时延敏感、缓存资源竞争强、端边协同链路复杂等特点。若仍采用静态缓存配置或单一调度策略，系统容易出现容器冷启动频繁、缓存命中率不足、任务排队时延上升、端边协同收益不稳定等问题。因此，需要设计并实现面向异构智能任务的操作系统弹性缓存技术软件，通过状态感知、到达预测、协同推断和动态缓存分配，支撑智能任务在多平台环境下的高效运行。

基于以上背景，系统需要围绕异构智能任务运行过程中的缓存保持、任务预测、端边协同和资源自适应分配建立完整的软件能力。本文档所描述的软件设计面向华为昇腾、飞腾等异构平台环境，当前系统实现覆盖四个课题的运行、展示和结果对比能力，其中第四个课题已开放执行设备、总轮数和请求强度配置，并通过吞吐量结果验证弹性缓存策略效果。需求分析分为以下四个部分：

基于冷、暖、热三状态的容器调度优化方法。异构智能任务在容器化运行时，不同容器实例会处于冷状态、暖状态和热状态。冷状态容器尚未加载模型与运行环境，启动时延较高；暖状态容器已完成部分环境初始化或缓存保留，能够降低启动开销；热状态容器已具备快速响应能力，适合承接高频任务。软件需要能够采集不同模型在冷、暖、热三状态下的启动时延、资源占用和任务完成时间，支持在飞腾、昇腾等平台上执行对比测试，并通过可视化方式展示状态调度优化对任务时延和资源利用率的影响。该部分对应当前系统的课题一，主要实现容器状态调度性能对比、模型选择、平台选择、任务执行和结果展示功能。

任务到达预测方法。异构智能任务的到达过程具有突发性、周期性和负载波动性。系统若不能提前感知任务到达趋势，将难以合理决定哪些模型应保持热状态、哪些缓存可以回收、哪些任务需要提前预热。软件需要基于历史任务到达数据、负载等级和数据集来源，执行任务到达预测分析，输出在低负载、高负载条件下的预测结果，并对不同平台、不同数据集组合进行对比展示。该部分对应当前系统的课题二，主要实现数据集分组选择、负载等级选择、板卡类型选择、预测任务执行以及结果文件解析展示。

端边协同推断机制。对于计算复杂度较高或端侧资源不足的智能任务，单端执行可能导致时延过高或资源占用过大。端边协同推断机制通过模型切分、端侧 head 执行、边侧 tail 执行以及中间特征传输，实现端侧与边侧之间的计算协作。软件需要支持昇腾服务器、昇腾客户端、飞腾客户端等角色组合，能够根据不同客户端类型加载对应切分配置，启动服务端推断进程和客户端推断进程，并展示各客户端的端到端时延、速度提升比以及共享切分缓存带来的内存节省。该部分对应当前系统的课题三，当前已支持单客户端场景和飞腾客户端加昇腾客户端的双客户端场景。

动态自适应缓存分配方法。随着任务到达模式、模型热度和设备资源状态持续变化，系统需要具备动态调整缓存分配策略的能力。该方法应综合考虑模型访问频率、容器状态、端边协同收益、内存占用和任务优先级，动态决定缓存保留、缓存迁移、缓存回收和预热对象。当前系统已开放第四个课题的运行入口，支持选择昇腾或飞腾执行设备、设置执行总轮数和请求强度，后端调用本地或远程缓存算法任务并解析吞吐量结果；缓存状态采集、策略基线对比和策略动作明细仍保留为后续扩展点。

### 1.2.2 系统组成

面向异构智能任务的操作系统弹性缓存技术软件由基于冷、暖、热三状态的容器调度优化模块（下文称“模块一”）、任务到达预测模块（下文称“模块二”）、端边协同推断模块（下文称“模块三”）、动态自适应缓存分配模块（下文称“模块四”）四个模块组成。当前系统已实现模块一至模块四的运行集成与结果展示，模块四已具备吞吐量实验入口，并在策略动作、基线吞吐量和收益明细方面保留扩展能力。

### 1.2.3 软件概述

本文档适用的系统为：面向异构智能任务的操作系统弹性缓存技术软件。

本文档适用的软件为：面向异构智能任务的操作系统弹性缓存技术软件。该软件运行于华为昇腾平台、飞腾平台及相关边缘计算环境中，支持通过前端界面发起课题运行任务，由后端接口调度本地脚本、远程服务器脚本和客户端脚本执行，并将执行日志和结果数据返回前端展示。

基于冷、暖、热三状态的容器调度优化模块的功能为：

1. 提供用户界面，允许用户选择硬件平台、运行模型和执行轮数；
2. 调用对应平台的容器状态测试脚本，执行冷、暖、热状态下的模型启动与推理测试；
3. 加载测试结果文件，并解析模型冷启动、暖启动和热状态运行指标；
4. 对比不同状态下的任务完成时延和资源利用情况；
5. 显示对比结果，包括端到端时延、内存占用和性能提升比等指标。

任务到达预测模块的功能为：

1. 提供交互式界面供用户选择板卡类型、数据集分组和负载等级；
2. 调用任务到达预测脚本，执行低负载或高负载条件下的预测任务；
3. 解析预测结果文件，提取不同平台上的预测精度、任务完成时间和资源使用情况；
4. 可视化展示任务到达预测在不同数据集和负载条件下的对比结果；
5. 支持华为昇腾和飞腾平台的运行结果加载与展示。

端边协同推断模块的功能为：

1. 提供用户界面，允许用户选择运行场景、模型、统计次数和服务端口；
2. 根据运行场景加载对应的模型权重、模型切分配置、客户端设备键和运行目录；
3. 远程启动昇腾服务器端推断进程，并根据场景启动飞腾客户端、昇腾客户端或双客户端；
4. 对客户端推断日志进行解析，获取端到端平均时延、服务端尾部推断时延和传输时延；
5. 在单客户端场景下展示单客户端速度提升结果，在双客户端场景下分别展示飞腾客户端和昇腾客户端速度提升结果，并展示共享切分缓存带来的内存节省。

动态自适应缓存分配模块的功能为已开放运行功能和扩展策略功能，当前实现目标为：

1. 提供用户界面，允许用户选择执行设备、执行总轮数和请求强度；
2. 在飞腾本机或昇腾远程节点调用缓存算法运行脚本；
3. 按设定请求强度驱动缓存调度器，采集任务执行吞吐量；
4. 输出执行设备、执行总轮数、请求强度、吞吐量、基线吞吐量和吞吐量提升比等结果字段；
5. 为后续缓存动作明细、预期时延收益、内存收益和策略执行状态展示保留数据结构。

### 1.2.4 运行环境

### 4.2.2 执行流程

基于冷、暖、热三状态的容器调度优化模块执行流程如下：

1. 用户选择课题一、硬件平台、运行模型和执行轮数。
2. 前端将参数提交至后端任务启动接口。
3. 后端校验平台、模型和轮数参数，生成受控执行命令。
4. 后端创建任务记录并启动脚本执行进程。
5. 脚本依次执行冷状态、暖状态和热状态容器调度测试，记录每轮任务指标。
6. 日志采集线程将执行日志写入任务日志队列，前端通过事件流实时展示。
7. 脚本完成后，后端读取结果文件并解析为统一结果表。
8. 前端展示结果表格和静态图表，并保留任务完成状态。

执行流程对应的逻辑关系为：参数输入组件负责形成实验约束；任务参数配置子模块负责校验和映射；容器状态调度执行子模块负责脚本调用；结果解析子模块负责结果格式转换；图表展示子模块负责面向用户展示状态调度收益。

硬件平台选择接口（IF-T1-PLATFORM-SELECT）：该接口由前端下拉选择组件提供，用户选择飞腾或昇腾后，前端将平台标识写入任务请求。后端根据平台标识选择本地执行路径或远程执行路径，避免用户直接输入脚本路径。

模型选择接口（IF-T1-MODEL-SELECT）：该接口用于选择待测试模型。模型列表由前端配置或后端配置生成，支持单模型执行和全部模型执行。后端对模型参数进行枚举校验，若模型不在允许列表内，则拒绝任务启动并返回错误信息。

执行轮数输入接口（IF-T1-ROUNDS-INPUT）：该接口用于控制重复测试次数。后端将轮数校验为正整数后传递至脚本，用于降低单次偶然波动对实验结果的影响。若输入为空、非数字或小于一，则任务启动失败。

任务启动接口（IF-T1-TASK-START）：该接口接收课题编号、平台、模型和轮数，返回任务ID。任务ID用于后续日志订阅、任务取消和结果查询。接口设计保证任务启动与结果展示解耦，支持长时间脚本执行过程中的实时进度反馈。

任务日志接口（IF-T1-TASK-LOG）：该接口以事件流方式返回脚本执行日志和任务状态。日志内容包括命令启动、状态切换、测试进度、结果保存和异常信息。日志接口不返回敏感凭据和不必要的系统路径细节。

结果文件接口（IF-T1-RESULT-FILE）：该接口由脚本执行进程生成，结果解析子模块读取。接口约束包括文件存在性、字段完整性和数值类型可解析性。若结果文件缺失，后端返回任务执行失败；若部分字段缺失，后端保留可解析字段并在日志中记录字段缺失情况。

### 5.2.2 执行流程

任务到达预测模块执行流程如下：

1. 用户选择课题二、板卡类型、数据集分组和负载等级。
2. 前端向后端任务启动接口提交预测任务参数。
3. 后端将板卡类型映射为内部板卡参数，将数据集分组映射为输入数据路径，将负载等级映射为低负载或高负载参数。
4. 后端校验参数均属于允许范围后，创建任务记录并启动预测脚本进程。
5. 预测脚本加载指定任务到达数据，执行任务到达预测并保存结果文件。
6. 日志采集线程持续采集数据加载、预测执行和结果保存信息。
7. 预测脚本完成后，结果解析子模块读取CSV结果文件并执行字段归一化。
8. 前端展示预测结果表格和负载对比图表。

该执行流程强调预测实验输入条件的一致性。板卡类型仅影响运行环境和平台结果，数据集分组和负载等级决定输入到达模式。通过统一解析结果，前端可以在不同板卡和不同数据集场景下采用一致的展示方式。

板卡类型选择接口（IF-T2-CARD-SELECT）：该接口用于限定预测任务的运行平台。前端以“昇腾”和“飞腾”形式展示，后端将其转换为脚本可识别参数。接口不允许用户输入任意主机名、脚本路径或设备路径。

数据集分组选择接口（IF-T2-DATASET-SELECT）：该接口用于选择任务到达数据来源。当前支持Google数据集和Huawei数据集两类分组，后端根据分组选择对应数据目录，并结合负载等级进一步确定数据切片。

负载等级选择接口（IF-T2-LOAD-SELECT）：该接口用于选择预测场景的负载水平。低负载场景用于验证稳定到达条件下的预测效果，高负载场景用于验证突发到达条件下的预测效果。后端通过枚举映射生成脚本参数，确保负载等级与数据切片一致。

预测任务启动接口（IF-T2-TASK-START）：该接口返回任务ID并启动预测脚本。接口应在任务启动前完成参数合法性检查、结果目录可访问性检查和脚本存在性检查。若检查失败，应返回明确错误信息并不启动子进程。

预测日志接口（IF-T2-TASK-LOG）：该接口用于向前端返回预测任务执行过程。日志内容包括参数映射、数据加载进度、预测算法运行状态、结果文件保存路径和错误原因。日志接口的主要目标是保证长时间预测任务具备可观察性。

预测结果文件接口（IF-T2-RESULT-FILE）：该接口由预测脚本输出CSV文件。结果解析子模块读取该文件并转换为统一结果结构。接口约束包括字段名称可识别、数值列可转换、至少存在一行有效结果数据。若结果为空，则返回预测任务无有效结果。

### 6.2.2 执行流程

端边协同推断模块执行流程如下：

1. 用户选择课题三、运行场景、模型、统计次数和服务端口。
2. 后端读取模型注册表，获得模型权重、结构名称、输入尺寸和切分配置。
3. 后端校验端口范围、统计次数和模型配置完整性。
4. 后端清理远程昇腾服务器上的残留server进程和端口占用。
5. 后端构建server启动命令，并在远程昇腾服务器上以 ONNX Runtime 模式启动tail推断服务。
6. 后端等待服务端口就绪，确认server已进入可连接状态。
7. 单客户端场景下，后端启动一个客户端推断进程；双客户端场景下，后端并发启动飞腾客户端和昇腾客户端推断进程。
8. 客户端执行head推断、特征传输、服务端tail推断调用和端到端时延统计。
9. 客户端完成后，后端解析日志并计算速度提升比。
10. 双客户端场景下，后端进一步计算共享切分缓存内存节省值和节省比。
11. 前端展示执行日志、速度提升结果和内存节省结果。

该执行流程以服务端先启动、客户端后连接为基本约束，避免客户端连接不存在的服务端口。双客户端场景通过并发执行反映多端共享切分缓存条件下的协同收益，但结果展示时仍分别保留飞腾客户端和昇腾客户端的指标，以便分析不同端侧设备的性能差异。

运行场景选择接口（IF-T3-SCENARIO-SELECT）：该接口用于选择端边协同角色组合。当前支持昇腾服务器与昇腾客户端、昇腾服务器与飞腾客户端、昇腾服务器与飞腾客户端加昇腾客户端三类场景。后端根据场景确定客户端数量、设备键和结果展示结构。

模型选择接口（IF-T3-MODEL-SELECT）：该接口由模型注册表提供可选项。用户只能选择注册表中存在的模型，后端根据模型名称查询权重路径、模型结构和切分配置。若模型配置不完整，则任务不启动。

统计次数输入接口（IF-T3-RUNS-INPUT）：该接口用于控制客户端推断统计样本数。为保证统计结果稳定，后端可在用户输入统计次数基础上叠加固定预热次数，但展示结果仅以正式统计次数计算平均时延。

服务端口输入接口（IF-T3-PORT-INPUT）：该接口用于指定server监听端口。后端校验端口为1024至65535之间的整数，并在server启动前执行端口占用检查或残留进程清理。

端边协同任务启动接口（IF-T3-TASK-START）：该接口接收端边协同运行参数并返回任务ID。任务启动后，后端先启动服务端，再启动客户端，任务完成后返回速度提升和内存节省结果。

远程命令执行接口（IF-T3-REMOTE-EXEC）：该接口用于在远程昇腾服务器或昇腾客户端上执行受控命令。命令由后端根据注册表和场景配置生成，接口不接受用户传入的任意命令字符串。

协同结果日志接口（IF-T3-LOG-PARSE）：该接口用于从服务端和客户端日志中提取性能指标。日志解析应识别端到端时延、服务端tail推断时延、传输时延、Baseline时延和Ours时延。若某项指标缺失，应在结果表中保留已解析指标，并在执行日志中说明缺失字段。

### 10.4.2 执行流程

动态自适应缓存分配模块当前执行流程如下：

1. 用户选择课题四、执行设备、执行总轮数和请求强度。
2. 前端向后端任务启动接口提交topicId、device、rounds和requestIntensity。
3. 后端校验执行设备、总轮数和请求强度范围。
4. 后端根据设备类型选择飞腾本地工作目录或昇腾远程工作目录。
5. 缓存算法执行子模块启动缓存调度器压测任务并采集日志。
6. 结果解析子模块从日志中读取__PROJECT4_RESULT__标记并解析吞吐量结果。
7. 前端展示执行日志、吞吐量结果表和topic4静态吞吐量对比图。

当前版本已执行吞吐量实验和结果展示，基线吞吐量、吞吐量提升比、缓存动作明细和策略下发流程作为后续扩展内容保留。
