# 数据格式与启动说明

本文说明课题图表演示模块的**配置文件格式**、**CSV 数据格式**，以及如何启动前端与后端服务。

---

## 一、整体结构

```
public/
├── config/
│   └── figures.json      # 课题 / 图表面板清单（仅元数据）
└── data/
    ├── topic1/
    │   ├── 图1：性能对比图.csv
    │   ├── 表1：昇腾测试数据.csv
    │   └── ...
    ├── topic2/
    └── ...
```

运行时流程：

1. 浏览器加载 `figures.json`，生成课题与图/表下拉框  
2. 按 `title` 定位数据文件：`public/data/topic{N}/{title}.csv`  
3. 前端解析 CSV，转为 ECharts 或 HTML 表格结构并渲染  

左侧「课题结果图表」走上述配置；右侧 K8s 监控面板走 Python 后端 API（可选）。

---

## 二、配置文件 `figures.json`

路径：`public/config/figures.json`

每个图/表条目**只写三个字段**，其余由程序按约定自动生成。

| 字段 | 类型 | 说明 |
|------|------|------|
| `topic` | 数字 | 课题编号，如 `1` 表示课题一 |
| `title` | 字符串 | 下拉框显示名称，且 **CSV 文件名与此完全一致** |
| `type` | 字符串 | 绘图类型，见下表 |

### 示例

```json
{
  "topics": [
    {
      "topic": 1,
      "figures": [
        { "title": "图1：性能对比图", "type": "mixed" },
        { "title": "表1：昇腾测试数据", "type": "table" },
        { "title": "表2：飞腾测试数据", "type": "table" }
      ]
    }
  ]
}
```

### `type` 取值说明

| type | 含义 | CSV 内容 |
|------|------|----------|
| `table` | 表格 | 首行表头 + 数据行（见 3.1） |
| `scatter` | 散点图 | 图表格式，所有序列均为散点 |
| `line` | 折线图 | 图表格式，所有序列均为折线 |
| `bar` | 柱状图 | 图表格式，所有序列均为柱状 |
| `pie` | 饼图 | 图表格式 |
| `mixed` | 混合图 | 图表格式，**必须写「类型」行** |

### 自动生成的规则（无需写入配置）

- **课题显示名**：`topic: 1` → 「课题一」  
- **数据文件**：`public/data/topic{编号}/{title}.csv`  
- **下拉框 id**：与 `title` 相同  

新增条目：在 `figures.json` 增加 `{ "title", "type" }`，并创建同名 CSV 文件。

## 三、CSV 数据格式

- 编码：**UTF-8**（导出脚本带 BOM，Excel 打开不乱码）  
- 编辑：Excel / WPS / VS Code 等  
- 含逗号、引号、换行的单元格用双引号包裹  

### 3.1 表格（`type: "table"`）

首行为列名，以下为数据行。

```csv
model,暖到热,冷到热,性能提升比
deit_tiny_patch_16_224,1.02,5.45,81.27
levit_128,1.15,5.02,77.06
```

### 3.2 图表（`scatter` / `line` / `bar` / `pie` / `mixed`）

自上而下：**可选元数据行 → 序列名表头行 → 数据行**（与原先 xlsx 布局相同，现统一存为 CSV）。

#### 元数据行（第一列标签，后续列为内容）

| 第一列 | 后续列 |
|--------|--------|
| `x轴` | 横轴名称 |
| `y轴` | 纵轴名称 |
| `类型` | 各序列绘图类型（多类型时必填） |

#### 序列名表头行

- 第一列：留空  
- 后续列：序列名称（图例）

#### 数据行

- 第一列：分类（X 轴类目）  
- 后续列：数值  

#### 示例（`图1：性能对比图.csv`）

```csv
x轴,模型与异构任务,,
y轴,性能提升比(%),,
类型,scatter,scatter,line
,昇腾性能提升比(%),飞腾性能提升比(%),KPI指标要求
deit_tiny_patch_16_224,81.27,70.44,40
levit_128,77.06,65.97,40
```

#### 「类型」列常用写法

| 值 | 效果 |
|----|------|
| `scatter` | 散点 |
| `line` / `line:symbol` | 折线 |
| `bar` | 柱状 |
| `bar:stack:Ad` | 堆叠柱状（组名 `Ad`，大小写保留） |
| `bar:emphasis` | 柱状高亮 |
| `pie` | 饼图 |

单一 `type`（如 `bar`）且各列类型一致时，可省略「类型」行。

## 五、服务启动

### 5.1 环境要求

- **Node.js** + **npm**  
- 课题图表仅依赖前端  
- K8s 面板需 **Python 3** + **kubectl**（可选）

### 5.2 安装与启动前端

```bash
cd /home/a/l-workspace/CPN-Frontend
npm install
npm run serve
```

默认端口 **7000**（占用时会自动换端口，以终端输出为准）。

内网访问推荐使用固定端口启动：

```bash
cd /home/a/l-workspace/CPN-Frontend
npm run serve:lan
```

该命令会将前端固定监听到 `0.0.0.0:7001`，可通过 `http://192.168.31.36:7001/` 从内网访问。

生产构建：`npm run build`（`dist/` 需包含 `config/` 与 `data/`）。

### 5.3 启动 K8s 后端（可选）

```bash
# 终端 1
cd /home/a/l-workspace/CPN-Frontend/server
python3 k3s-api.py

# 终端 2
cd /home/a/l-workspace/CPN-Frontend
npm run serve
```

API 端口 **8099**，前端 `/api` 代理到该地址。

也可以在项目根目录直接启动后端：

```bash
cd /home/a/l-workspace/CPN-Frontend
npm run serve:backend
```

若使用 `http://192.168.31.36:7001/` 访问页面，必须同时保持后端 `8099` 运行，否则 `/api` 请求会失败。
